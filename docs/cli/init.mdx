title: "init"
description: "Interactive setup wizard for configuring your agent team"
---

Interactive setup wizard that configures your cloud provider, instance type, owner info, and agent team. Writes `clawup.yaml` and `.env.example` — no secrets are prompted during init.

## Usage

```bash
clawup init [options]
```

## Options

| Option | Description |
|--------|-------------|
| `-y, --yes` | Skip confirmation prompts |

## Two Modes

### Fresh Init (no `clawup.yaml`)

Full interactive wizard that walks you through all configuration:

1. **Stack Name** — unique deployment name (e.g., `dev`, `prod`)
2. **Cloud Provider** — AWS or Hetzner
3. **Region** — deployment region
4. **Instance Type** — server size (affects cost)
5. **Owner Configuration** — name, timezone, working hours
6. **Agent Selection** — built-in identities, Git URL, local path, or mix
7. **Template Variables** — custom values used by identity workspace files
8. **Review & Confirm** — full config summary with cost estimates

### Repair Mode (existing `clawup.yaml`)

When `clawup.yaml` already exists, init enters **repair mode** — a non-interactive refresh that:

1. Re-fetches all agent identities (picks up upstream changes)
2. Updates the secrets section with any new plugins, deps, or `requiredSecrets`
3. Refreshes inline plugin config with latest `pluginDefaults` from identities
4. Prompts only for **new** template variables not already in the manifest
5. Regenerates `.env.example` to reflect current requirements

Existing manifest values (stack name, provider, region, agents, etc.) are preserved. This is useful after updating an identity repo or adding new plugins to an identity.

## Examples

```bash
# First-time setup
clawup init

# Refresh after identity changes
clawup init
```

## Output

Creates (or updates) in your project directory:
- `clawup.yaml` — deployment manifest with `${env:VAR}` secret references
- `.env.example` — all required env var names with empty values

The `.clawup/` directory and `.env` are added to `.gitignore` automatically.

## Next Steps

After `init` completes:

1. Copy `.env.example` to `.env` and fill in your secrets
2. Run [`clawup setup`](/cli/setup) to validate secrets and configure Pulumi
3. Run [`clawup deploy`](/cli/deploy) to deploy your agents

See [Environment Variables](/configuration/environment) for full details on the `${env:VAR}` reference system.
